What is an AddOn in Hybris?

AddOns in SAP Commerce are a powerful mechanism for extending and customizing Hybris without impacting the core functionality. They offer a modular, flexible, and upgrade-safe way to add new features and integrate third-party systems into the platform. By using AddOns, businesses can build a tailored e-commerce experience while maintaining the integrity and maintainability of the SAP Commerce solution.

- Built on top of the existing SAP Hybris Commerce to extend the functionality of the SAP Hybris Commerce Accelerator.

- Addons are a form of extension that allows you to add front-end files e.g. JSP, CSS, and JS via your AddOn. (May or may not include frontend )

- No editing in core code of SAP Hybris (This is considered as best practice)

- Adhere to best practices in SAP Hybris development.

- Safely upgrade your core SAP Hybris code without the need for excessive development.

- You can add and remove the AddOn, whenever needed without refactoring or otherwise rewriting code.

In SAP Commerce (Hybris), an AddOn is a type of extension that allows developers to add new functionalities to an existing storefront (or other parts of the SAP Commerce application) without directly modifying the core code or the original extension code. AddOns are typically used to extend the default functionality of the Hybris platform, specifically in storefronts built on Accelerators, such as adding new features or customizing the storefront’s user interface.

An AddOn is a modular piece of code that can be injected into a Hybris solution without altering the base system's code. This provides flexibility in adding new features to the platform without disturbing the integrity of the original system, allowing you to customize and extend the behavior of the Hybris application easily.

Structure of AddOns

AddOns are usually packaged as separate extensions that follow specific guidelines for integration into the existing Hybris platform. The typical structure of an AddOn includes:

  • Frontend code: HTML, JavaScript, and CSS for the storefront.
  • Backend logic: Java classes or services that interact with the Hybris service layer, possibly modifying behavior or adding new functionality.
  • Integration hooks: These hooks are points where the AddOn integrates with the storefront or other modules.
  • Resources: Configuration files, templates, or other assets that the AddOn uses.

How AddOns Work

  1. Modular Extension: An AddOn is installed as an extension but does not require modifying the core or any other extension code directly.
  2. Integration with Accelerators: AddOns are commonly used with Hybris Accelerators to inject custom behavior into a storefront. For example, you could create an AddOn to add a payment gateway to an accelerator-based storefront.
  3. Seamless Upgrades: Since AddOns don’t modify core files or logic, they are usually upgrade-safe. You can update or upgrade Hybris without worrying about breaking customizations made through AddOns.
  4. Dynamic Behavior: The AddOn might include components, services, or even front-end resources that are dynamically injected into the application depending on the business needs.

Example Use Cases for AddOns

  1. Payment Gateway Integration: You can create an AddOn to integrate a custom payment gateway without modifying core Hybris components.
  2. Custom Search: An AddOn can be used to integrate a custom search solution (like Elasticsearch or a specific recommendation engine) into the storefront.
  3. Custom Widgets: AddOns can add new widgets to the storefront that provide extra functionality, such as banners, promotional modules, or custom product recommendations.
  4. Third-Party Integrations: AddOns are ideal for integrating external systems like CRM, ERP, or other marketing tools without altering Hybris’s core functionality.

Advantages of Using AddOns

  1. Separation of Concerns:
    • AddOns help keep your core Hybris system code clean and maintainable by isolating custom functionality into modular extensions. This makes it easier to manage and maintain your custom features.
  2. Flexibility:
    • You can add new features or change existing functionality on the fly without modifying the core codebase. This means you can customize your SAP Commerce environment to meet specific business needs without touching the base platform’s code.
  3. Upgrade Compatibility:
    • Since AddOns do not alter the core code, they tend to be more upgrade-safe. When you upgrade SAP Commerce, your custom AddOns are less likely to break, provided they follow good coding practices.
  4. Modular Development:
    • AddOns can be developed and tested independently, making them easier to deploy and scale. This modularity allows for focused development, which can result in faster time to market for new features or integrations.
  5. Reduced Risk:
    • By using AddOns, you avoid directly modifying the core extensions or platform logic, which can reduce the risk of introducing bugs or incompatibilities with future Hybris upgrades.
  6. Reusability:
    • AddOns can be reused across different Hybris projects or environments. For example, you can create an AddOn that handles a specific payment method and reuse it across different stores or instances of SAP Commerce.
  7. Customizable and Extensible:
    • AddOns can be customized and extended to meet specific business requirements. This allows businesses to implement unique features without being constrained by the out-of-the-box functionality.

Example of an AddOn Creation

To create an AddOn in SAP Commerce, you typically:

  1. Define your AddOn in the Hybris localextensions.xml.
  2. Develop the AddOn by following the structure required for Hybris extensions (such as adding services, controllers, and frontend resources).
  3. Include necessary templates, configuration, and resource files to integrate your AddOn into the storefront or back-office. 
        <extension name="myaddon" />
